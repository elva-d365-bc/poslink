    POS Retail Connect — SharePoint / OneDrive integrācija

    1. Pārskats

    POS Retail Connect nodrošina kopēju SharePoint / OneDrive failu apmaiņas mehānismu, kuru var izmantot jebkurš POS Retail Connect modulis gan failu nosūtīšanai (augšupielāde uz SharePoint/OneDrive), gan failu saņemšanai (lejupielāde no SharePoint/OneDrive). Piemērs: POS Retail Connect Scale modulis augšupielādē svaru grupas failus; citi moduļi var lejupielādēt failus, ko ārējās sistēmas ir novietojušas noteiktā mapē.

    Savienojumi tiek konfigurēti centralizēti lapā SharePoint Connections un katrs modulis savos uzstādījumos norāda izmantojamā savienojuma kodu.

    Autentifikācija notiek caur Microsoft Graph API ar OAuth2 Client Credentials plūsmu (lietotne-uz-lietotni, bez lietotāja iesaistīšanas, bez Refresh Token). Izmantojot Client Credentials, tokens tiek iegūts tieši no Azure AD ar Client ID un Client Secret — nav nepieciešama lietotāja autorizācija vai tokenu atjaunošanas plūsma.

    2. Azure AD lietotnes reģistrācija

    2.1 Reģistrācija

    1. Dodieties uz portal.azure.com → Microsoft Entra ID → App registrations → New registration.
    2. Nosaukums: **BC ELVA Retail Integration** (vai cits uzņēmumam nozīmīgs nosaukums).
    3. Account types: Accounts in this organizational directory only.
    4. Register.

    2.2 Klienta noslēpums (Client Secret)

    1. Certificates & secrets → New client secret.
    2. Apraksts: **BC ELVA Retail Integration Secret**.
    3. Derīguma termiņš: 24 mēneši (maksimālais standarta termiņš).

      ⚠️ Uzreiz pēc izveidošanas iestatiet Calendar / Outlook atgādinājumu 23 mēnešu laikā. Pēc termiņa beigām failu apmaiņa nedarbosies!

    4. Nokopējiet noslēpuma Value — tas redzams tikai vienu reizi.
    5. Dynamics 365 Business Central uzstādījumos: ievadiet Client ID un Secret, aizpildiet Client Secret Expiry Date lauku ar reālo beigu datumu.

    Vērtīgs padoms: Dynamics 365 Business Central brīdinās, ja noslēpuma termiņš ir mazāks par 30 dienām, un prasīs lietotāja apstiprinājumu turpināt failu apmaiņu, ja termiņš jau beidzies.

    2.3 API atļaujas

    Atļauja Tips Kāpēc vajadzīga
    **Files.ReadWrite.All** Application (nevis Delegated) OneDrive failu lasīšanai un rakstīšanai
    **Sites.ReadWrite.All** Application (nevis Delegated) SharePoint failu lasīšanai un rakstīšanai

    ⚠️ Kritisks solis: Pēc atļauju pievienošanas ir obligāti jānoklikšķina Grant admin consent for [jūsu organizācija].
    Blakus katrai atļaujai jāparādās zaļai atzīmei ar tekstu "Granted for...".
    Ja atzīme nav zaļa — augšupielāde neizdosies ar 401 kļūdu.

    ⚠️ Svarīgi: Atļaujām jābūt Application tipam, nevis Delegated. Kļūda "Unsupported app only token" nozīmē, ka atļaujas ir uzstādītas kā Delegated.

    3. OneDrive — pakalpojuma konts

    3.1 Izmantojiet speciālu pakalpojuma kontu

    Neizmantojiet parasta darbinieka e-pasta adresi laukā "OneDrive User Email". Ja darbinieks mainās un tā konts tiek slēgts — integrācija nedarbojas.

    Ieteicamais risinājums:

    • Izveidojiet speciālu Microsoft 365 pakalpojuma kontu, piem., **retailintegration@contoso.onmicrosoft.com**.
    • Piešķiriet tam SharePoint/OneDrive licenci (nepieciešama, lai OneDrive būtu aktivizēts).
    • Šim kontam ir jāpieslēdzas OneDrive vismaz vienu reizi, lai tiktu inicializēta krātuve.
    • Dokumentējiet šo kontu IT sistēmu sarakstā.

    Alternatīva: Izmantojiet SharePoint dokumentu bibliotēku (nevis OneDrive) — tad personīgā konta problēma nebūs.

    4. SharePoint — vietnes priekšnosacījumi

    4.1 Vietne jāizveido pirms Dynamics 365 Business Central konfigurācijas

    Dynamics 365 Business Central neizveido SharePoint vietnes automātiski. Vietnei jābūt izveidotai iepriekš, jo SharePoint vietnes izveide ir nozīmīga IT administratīva darbība — tā izveido Microsoft 365 grupu, pastkastes, kalendārus, Teams kanālus u.c.

    Soļi pirms Dynamics 365 Business Central konfigurācijas:

    1. SharePoint admin atvērt SharePoint Admin Center → Sites → Active sites → Create.
    2. Izvēlēties vietnes veidu (piemēram, Team site vai Communication site).
    3. Nosaukums, URL, īpašnieks.
    4. Pēc izveides pārliecināties, ka vismaz viena dokumentu bibliotēka ir redzama.

    Ieteicamais URL piemērs: **https://contoso.sharepoint.com/sites/RetailIntegration** ()

    Piezīme: Mapes ceļš (gan augšupielādei, gan lejupielādei) tiek konfigurēts katra moduļa uzstādījumos, nevis kopējā savienojuma konfigurācijā.

    4.2 Piekļuves prasības

    Azure AD lietotnei (**BC ELVA Retail Integration**) nav nepieciešams kļūt par vietnes dalībnieku — **Sites.ReadWrite.All** Application atļauja piešķir piekļuvi visām vietnēm.

    5. Azure AD lietotnes reģistrācija Dynamics 365 Business Central

    Lai Dynamics 365 Business Central varētu izmantot izveidoto App Registration (no 2.–3. soļa), klienta ID jāreģistrē arī Dynamics 365 Business Central pusē — lapas Microsoft Entra Applications (iepriekš: AAD Applications) sarakstā.

    5.1 Pievienošana Dynamics 365 Business Central

    1. Dynamics 365 Business Central meklējiet Microsoft Entra Applications
    2. Nospiediet New
    3. Aizpildiet:
    Lauks Vērtība
    Client ID Ielīmējiet Application (client) ID no Azure AD
    Description Piem., **ELVA Retail BC Integration**
    State **Enabled**
    1. Nospiediet Grant Consent — tas izveido service principal jūsu tenantā un saista to ar Dynamics 365 Business Central
    2. Sadaļā User Permission Sets pievienojiet katra moduļa Permission Set, kuram nepieciešama piekļuve Dynamics 365 Business Central API.

    Pievienojiet tikai tās tiesibas, kuras reāli tiek izmantotas. Minimāli vajadzīga lasīšanas/rakstīšanas piekļuve tiem Dynamics 365 Business Central API resursiem, kurus izmanto attiecīgais modulis.

    5.2 Kāpēc abi soļi ir nepieciešami

    Azure AD App Registration (2.–3. solis) piešķir tiesības Azure AD pusē (Microsoft Graph izsaukumiem no Dynamics 365 Business Central). Registrēšana Dynamics 365 Business Central Entra Applications pusē piešķir tiesības Dynamics 365 Business Central pusē (ārējais modulis vai sistēma var izsaukt Dynamics 365 Business Central API). Abi soļi jāizpilda.

    6. Ārējo aģentu — lasīšanas piekļuve

    ⚠️ Šis punkts attiecas uz ārējiem aģentiem vai sistēmām, kas lejupielādē failus no SharePoint/OneDrive (piemēram, POS Retail Connect Scale aģents, kas ielasa svaru failus; vai jebkurš cits POS Retail Connect modulis, kurš saņem failus no ārējās sistēmas).

    6.1 Problēma

    Dynamics 365 Business Central augšupielādē vai novieto failus SharePoint/OneDrive ar **Sites.ReadWrite.All** / **Files.ReadWrite.All**. Ārējā sistēma vai Dynamics 365 Business Central modulis, kurš nolasa šos failus, var izmantot to pašu savienojumu vai atsevišķu.

    6.2 Risinājums

    Ir divas iespējas:

    A) Tas pats Azure AD app + tās pašas atļaujas:

    • Konfigurējiet aģentu ar tiem pašiem Client ID un Secret.
    • **Files.ReadWrite.All** / **Sites.ReadWrite.All** ietver arī lasīšanas tiesības.

    B) Atsevišķs lasīšanas konts (ieteicams drošībai):

    • Izveidojiet otru App Registration, piem., **BC ELVA Retail Reader**.
    • Piešķiriet tikai **Files.Read.All** vai **Sites.Read.All** (Application).
    • Konfigurējiet aģentu ar šī otrā app Client ID un Secret.

    Princips "mazāk atļauju — mazāks risks" — ja aģenta akreditācijas dati tiek nopludināti, var tikai lasīt, nevis rakstīt.

    7. Dynamics 365 Business Central konfigurācija

    7.1 Atļaut HttpClient pieprasījumus

    Pirms jebkurš POS Retail Connect paplašinājums var izsaukt ārējos pakalpojumus (Graph API), jāiespējo izejošie HTTP pieprasījumi katram paplašinājumam, kas izmanto SharePoint/OneDrive integrāciju:

    1. Dynamics 365 Business Central atveriet Extension Management (Paplašinājumu pārvaldību)
    2. Atrodiet attiecīgo paplašinājumu (piemēram, POS Retail Connect Scale, vai jebkuru citu moduli, kas veic failu apmaiņu)
    3. Nospiediet trīs punktus → Allow HttpClient Requests → iestatiet Yes

    Šis iestatījums jāiespējo katram paplašinājumam, kas veic HTTP izsaukumus. Bez tā visi HTTP izsaukumi neizdosies bez kļūdas ziņojuma.

    Pēc aizpildīšanas izmantojiet pogu Test Connection, lai pārbaudītu savienojumu.

    7.3 Moduļa uzstādījumi — savienojuma piesaiste

    Katrs modulis savos uzstādījumos norāda, kuru savienojumu izmantot, ar lauku SharePoint Connection Code. Mapes ceļš un citi moduļa specifiskie parametri (piemēram, augšupielādes vai lejupielādes mape, arhīva mape) tiek konfigurēti katra moduļa uzstādījumos, nevis kopējā savienojumā.

    Tipiskais princips:

    • Eksportam (Dynamics 365 Business Central → SharePoint/OneDrive): modulis raksta failu norādītajā mapē.
    • Importam (SharePoint/OneDrive → Dynamics 365 Business Central): modulis nolasa failus no norādītās mapes un pēc apstrādes pārvieto tos uz arhīva mapi.
    Modulis Uzstādījumu lapa Moduļa specifiskie lauki
    POS Retail Connect Scale Scale Integration Setup (Device Group) SharePoint Connection Code, SharePoint Folder Path
    Citi moduļi Katra moduļa Setup lapa SharePoint Connection Code, mapes ceļi atbilstoši moduļa loģikai

    10. Ātrs pārbaudes algoritms (TestConnection)

    Savienojuma lapā SharePoint Connection ir pieejama poga Test Connection. Tā veic 5 soļus:

    1. Iegūst OAuth2 tokenu (pārbauda Client ID + Secret).
    2. Pārbauda, vai tokens ir derīgs Graph API (**GET /organization**).
    3. Pārbauda Graph API atļaujas (**GET /sites/root**).
    4. Atrisina diskdziņa URL (atrod SharePoint vietni / OneDrive lietotāju).
    5. Pārbauda, vai mērķa mape eksistē (informatīvi — mape tiks izveidota automātiski).
